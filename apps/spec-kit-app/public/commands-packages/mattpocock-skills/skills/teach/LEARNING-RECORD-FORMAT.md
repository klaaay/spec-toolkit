# 学习记录格式

学习记录保存在 `./learning-records/`，并使用顺序编号：`0001-slug.md`、`0002-slug.md`，依此类推。目录按需创建，只在写入第一条记录时创建。

它们是教学中的 ADR：记录不明显的经验、关键洞见和用户声明过的既有知识，用来指导未来课程。它们也用于计算用户的最近发展区。

## 模板

```md
# {学到或确认的内容的短标题}

{1-3 句话：学到了什么，或确认了什么既有知识；说明它为什么会影响后续课程。}
```

格式到这里就结束。一条学习记录可以只有一个段落。它的价值在于记录 “这件事现在已知” 以及 “它为什么改变接下来该教什么”，而不是填满一堆小节。

## 可选小节

只有在确实有价值时才加入。大多数记录不需要这些小节。

- **Status** frontmatter（`active | superseded by LR-NNNN`）：当早期理解后来被证明错误并被替换时有用。
- **Evidence**：用户如何展示了理解，例如答对问题、完成练习、说明既有经验。当这个判断未来可能被重新审视时有用。
- **Implications**：这会解锁什么，或排除哪些后续课程方向。只有在影响不明显时才值得记录。

## 编号

扫描 `./learning-records/`，找到当前最高编号，然后加一。

## 什么时候写学习记录

满足以下任一条件时，写一条记录：

1. **用户展示了对某个非平凡内容的真实理解**，不只是接触过，而是有证据说明他们能正确使用这个概念。这会设定后续课程的新起点。
2. **用户透露了既有知识**，例如 “我已经知道 X”。记录下来，避免未来重复教学。同时记录他们声称的掌握深度。
3. **纠正了一个误解**。用户之前相信某个错误说法，现在理解了为什么错。这类记录很有价值，因为它能预测相关主题未来的卡点。
4. **使命因学习而变化**。用户发现自己真正关心的事和一开始想的不一样。交叉引用 [[MISSION.md]] 并更新它。

### 什么不算学习记录

- 只是覆盖过的材料。覆盖不等于学习。等待证据。
- 已经在 [[GLOSSARY.md]] 中用术语定义精炼记录的内容。不要重复。
- 按会话流水记录的活动日志。学习记录不是日记，而是足以影响决策的洞见。

## 替代与废止

如果后续记录和早期记录冲突，例如用户理解变深或被纠正，不要删除旧记录。把旧记录标记为 `Status: superseded by LR-NNNN`。理解如何演化，本身也是有用信号。
