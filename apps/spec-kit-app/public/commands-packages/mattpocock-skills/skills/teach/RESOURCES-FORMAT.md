# RESOURCES.md 格式

`RESOURCES.md` 是当前主题的可信来源集合。讲解中的知识应来自这里，而不是来自模型参数中的猜测。智慧来自这里列出的社区。

## 结构

```md
# {主题} 资源

## 知识

- [Book: _The Science and Practice of Strength Training_ — Zatsiorsky & Kraemer](https://example.com)
  关于训练编排和适应的基础文本。适用：周期安排、恢复、强度区间等内容。
- [Article: "How Much Should I Train?" — Greg Nuckols (Stronger By Science)](https://example.com)
  关于训练容量标志点的循证综述。适用：每个肌群每周训练组数目标。

## 智慧（社区）

- [r/weightroom](https://reddit.com/r/weightroom)
  高信噪比 subreddit，版主管理严格，反对伪科学。适用：训练计划评审、平台期排查。
- Local: {健身房名称} 周二力量课
  适用：获得关于动作的实时教练反馈。
```

## 规则

- **只收高可信来源。** 优先选择一手资料、受认可的专家、同行评审研究，以及管理严格的社区。如果某个资源只是披着教育外衣的营销内容，就不要收录。
- **每条都要注释。** 三个月后，裸链接没有价值。为每条资源加一行：它覆盖什么，以及什么时候该用它。
- **按知识 / 智慧分组。** 这对应 [SKILL.md](./SKILL.md) 中的教学理念。一个资源只出现在其中一组也可以。
- **明确暴露缺口。** 如果使命需要某个方向，但没有好资源，写一个 `## Gaps` 小节列出缺失内容。这会驱动未来搜索。
- **坚决修剪。** 如果某个资源后来被证明错误、浅薄或偏离使命，应删除它，而不是埋在列表里。五个锋利来源胜过三十个平庸来源。
- **记录社区偏好。** 如果用户选择不加入社区，在这里记下，避免未来会话反复提议。
