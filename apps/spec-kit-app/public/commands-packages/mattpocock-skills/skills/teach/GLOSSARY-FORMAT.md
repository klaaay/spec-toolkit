# GLOSSARY.md 格式

`GLOSSARY.md` 是这个教学工作区的规范语言。所有讲解、练习和学习记录都应遵循其中的术语。构建术语表本身也是学习的一部分：把一个概念压缩成精炼定义，说明用户已经理解了它。

## 结构

```md
# {主题} 术语表

{用一两句话说明这个术语表覆盖的主题。}

## 术语

**肌肥大**：
由多次训练中的机械张力和代谢压力驱动的肌肉增长。
_避免使用_：增肌、变壮

**渐进超负荷**：
随时间系统性提高肌肉承受的需求，可以通过重量、容量或强度实现。
_避免使用_：更努力、升级

**RPE（主观用力程度）**：
用 1-10 分自评一组动作有多费力。10 表示力竭，8 表示还能再做两次。
_避免使用_：努力分数、强度评分
```

## 规则

- **只有当用户已经理解某个术语时，才把它加入术语表。** 术语表是压缩知识的记录，不是用户用来第一次学习的词典。如果用户刚接触一个概念，等到他们能正确使用它后，再把它提升为术语。
- **要有立场。** 当同一个概念有多个说法时，选择最好的一个，并把其他说法列为应避免的别名。这就是语言压缩的方式。
- **定义要紧。** 一两句话即可。定义这个术语是什么，而不是它做什么或如何操作。
- **在定义中使用术语表自己的术语。** 一旦术语进入术语表，就优先在所有地方使用它，包括其他定义。这会让复杂术语之后更容易理解。
- **自然出现分组时，用小标题组织。** 例如 `## 解剖`、`## 编程`。如果术语彼此连贯，平铺列表也可以。
- **明确标记歧义。** 如果某个术语在外部领域中用法宽泛，写出本工作区的判定。例如：“在这个工作区中，‘组’ 始终指正式工作组，热身组单独记录。”
- **随着理解加深持续修订。** 用户第一周写下的定义，到第六周可能已经不准确。直接更新原条目，不要留下过期内容。
