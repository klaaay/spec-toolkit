# Automation 命令包工作流

本文档说明 `automation` 命令包提供的技能及其用途。

## 概述

`automation` 沉淀与浏览器自动化相关、可复用的 Agent 能力。当前以 **PinchTab** 为主：通过稳定的无障碍树引用（如 `e5`）、低 token 的正文抽取，以及持久化用户档案或多实例，让 Agent 能可靠地驱动 Chrome 完成真实站点上的任务。

## 当前技能

- **`pinchtab`**
  - **作用**：优先用 `pinchtab` CLI 完成导航、快照、点击、填写、按键等操作；需要管理档案或无法执行 shell 时，再用 `curl` 调用 HTTP API。
  - **适用**：需要打开网页、走登录/表单流程、抓取页面文字、多站点并行实例、导出截图或 PDF；希望比纯截图更省上下文、引用稳定时使用。

- **配套参考**（均在 `pinchtab/references/` 与 `TRUST.md`）
  - API、CLI、环境变量、档案管理、MCP 工具说明，以及安全信任与 Agent 优化建议。

## 使用建议

1. 先确认本机已安装 Chrome/Chromium 与 `pinchtab`，并按任务选择默认 `http://localhost` 或用户提供的远程地址与 Token。
2. 执行任务前选定合适的实例或用户档案（公开爬取用临时实例，重复登录站点用命名档案）。
3. 导航后先用 `snap -i -c` 或 `text` 观察页面，仅在页面结构变化后重新快照，避免使用过期引用 `e#`。
4. 需要保密的场景请阅读 `TRUST.md`，远程暴露服务时务必启用 Token 并使用专用自动化档案。
