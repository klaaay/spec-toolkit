# 设计文档审查者提示模板

在分派设计文档审查子代理时使用此模板。

**目的：** 验证 spec 是否完整、一致，并且足够清晰，可以进入实施规划阶段。

**适用时机：** 设计文档已经写入 `docs/superpowers/specs/`

```
任务工具（通用目的）:
  description: "审查设计文档"
  prompt: |
    你是一名设计文档审查者。请确认这份 spec 是否完整，并且已经准备好进入实施规划阶段。

    **待审查的 spec：** [SPEC_FILE_PATH]

    ## 检查项

    | 类别 | 关注点 |
    | --- | --- |
    | 完整性 | TODO、占位符、TBD、缺失段落 |
    | 一致性 | 内部矛盾、彼此冲突的要求 |
    | 清晰度 | 是否存在足以导致实现方向跑偏的歧义 |
    | 范围 | 是否足够聚焦，能支撑单份实施计划 |
    | YAGNI | 是否夹带未被请求的功能或过度设计 |

    ## 校准原则

    **只标出会在实施规划阶段造成真实问题的事项。**
    轻微措辞优化、风格偏好、某节比另一节更简略，这些都不构成阻塞。

    只有当存在严重缺口、明显矛盾、占位内容或会导致错误实施的歧义时，才不予通过。

    ## 输出格式

    ## Spec Review

    **状态：** 已通过 | 发现问题

    **问题（如有）：**
    - [Section X]: [具体问题] - [为什么会影响规划]

    **建议（不阻塞通过）：**
    - [改进建议]
```
