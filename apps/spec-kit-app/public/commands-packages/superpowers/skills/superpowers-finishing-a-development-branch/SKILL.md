# Finishing a Development Branch

## 概览

当实现完成且测试通过时，保持分支原样，保留工作树。

**核心原则：** 验证测试 → 保持分支原样 → 保留工作树。

**开始时宣布：** "我正在使用 `skills/superpowers-finishing-a-development-branch` skill 来完成这项工作。"

## 流程

### 第 1 步：验证测试

**在完成之前，验证测试通过：**

```bash
# 运行项目测试套件
npm test / cargo test / pytest / go test ./...
```

**如果测试失败：**
```
测试失败（<N>个失败）。必须在完成前修复：

[显示失败]

测试通过前无法完成。
```

停止。不要继续到第 2 步。

**如果测试通过：** 继续到第 2 步。

如果项目没有测试，忽略此步骤，直接进入第 2 步。

### 第 2 步：保持原样

实施完成。保持分支原样（稍后处理）。

报告："保持分支 <name>。工作树保留在 <path>。"

**不要清理工作树。**


## 快速对照

| 选项     | 合并 | 推送 | 保留工作树 | 清理分支 |
| -------- | ---- | ---- | ---------- | -------- |
| 保持原样 | -    | -    | ✓          | -        |

## 常见错误

**跳过测试验证**
- **问题：** 未验证测试就完成工作
- **修复：** 先验证测试

**清理工作树**
- **问题：** 误删工作树
- **修复：** 不要清理工作树，保持原样

## 红旗

**绝不要：**
- 测试失败仍继续
- 清理工作树

**务必：**
- 先验证测试
- 保持分支原样
- 保留工作树
