# 分层防御式校验

## 概览

当 bug 是由无效数据引起时，只在一个点加校验通常不够。  
不同调用路径、重构、mock 或环境差异，都可能绕过单点校验。

**核心原则：** 数据流经的每一层都校验，把 bug 变成结构上不可能发生的事。

## 四层防线

1. **入口校验**
   - 在 API 或边界层拒绝显然无效的输入

2. **业务逻辑校验**
   - 确认数据对当前操作来说仍然有意义

3. **环境防护**
   - 在危险环境里加保险，例如测试环境禁止对真实路径做危险操作

4. **调试埋点**
   - 一旦前三层还失守，至少能留下上下文证据

## 什么时候用

当你已经找到根因，而且该根因涉及：
- 空值 / 非法值传播
- 错目录 / 错路径 / 错环境
- 调用路径太多，单点防护不可靠

## 操作方式

1. 先画出数据流
2. 标出数据经过的每个关键检查点
3. 每层都加校验
4. 逐层测试，确保前一层被绕过时，后一层仍能兜住
