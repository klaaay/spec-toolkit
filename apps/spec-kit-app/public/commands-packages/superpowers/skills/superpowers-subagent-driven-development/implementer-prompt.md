# 实现者子代理提示模板

在分派实现者子代理时使用此模板。

```
任务工具 (通用目的):
  description: "实现任务 N: [任务名称]"
  prompt: |
    您要完成的任务是：任务 N: [任务名称]

    ## 任务描述

    [计划中任务的完整文本 - 请直接粘贴在此处，不要让子代理读取文件]

    ## 上下文

    [场景设置：该任务在整体中的位置、依赖关系、架构上下文]

    ## 开始之前

    如果您对以下方面有疑问：
    - 需求或验收标准
    - 方法或实现策略
    - 依赖关系或假设
    - 任务描述中任何不清楚的内容

    **请立即提问。** 在开始工作前，请提出所有疑虑。

    ## 您的工作

    一旦您明确了需求：
    1. 严格按照任务要求实现
    2. 编写测试（如任务要求，请遵循 TDD）
    3. 验证实现是否正常工作
    4. 提交您的工作成果
    5. 进行自审查（见下文）
    6. 汇报工作进展

    工作目录：[目录]

    **工作过程中：** 如遇意外或不明确的情况，**请及时提问**。
    随时可以暂停并澄清。不要猜测或做假设。

    ## 报告前：自审查

    以全新视角审视您的工作。自问：

    **完整性：**
    - 是否完全实现了规范中的所有内容？
    - 是否遗漏了任何需求？
    - 是否存在未处理的边缘情况？

    **质量：**
    - 这是我能做到的最佳工作吗？
    - 命名是否清晰准确（反映功能而非实现方式）？
    - 代码是否整洁且易于维护？

    **纪律：**
    - 是否避免了过度构建（YAGNI）？
    - 是否仅构建了请求的内容？
    - 是否遵循了代码库中的现有模式？

    **测试：**
    - 测试是否真正验证了行为（而非仅模拟）？
    - 如需要，是否遵循了 TDD？
    - 测试是否全面？

    如在自审查中发现任何问题，请先修复再报告。

    ## 报告格式

    完成后，请报告：
    - 实现的内容
    - 测试的内容及结果
    - 修改的文件
    - 自审查发现（如有）
    - 任何问题或疑虑
```
