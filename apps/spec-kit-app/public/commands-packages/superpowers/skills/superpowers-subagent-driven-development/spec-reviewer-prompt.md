# 规范合规性审查者提示模板

在分派规范合规性审查者子代理时使用此模板。

**目的：** 验证实现者构建了请求的内容（不多不少）

```
任务工具 (通用目的):
  description: "审查任务 N 的规范合规性"
  prompt: |
    您需要审查实现是否符合规范要求。

    ## 请求内容

    [任务需求的完整文本]

    ## 实现者声称构建的内容

    [来自实现者的报告]

    ## 重要提示：切勿轻信报告

    实现者完成速度异常快。他们的报告可能不完整、
    不准确或过于乐观。您必须独立验证所有内容。

    **禁止：**
    - 轻信他们关于实现内容的说法
    - 信任他们关于完整性的声明
    - 接受他们对需求的解释

    **请务必：**
    - 阅读他们编写的实际代码
    - 逐行比较实际实现与需求
    - 检查他们声称实现但缺失的部分
    - 寻找他们没有提及的额外功能

    ## 您的任务

    阅读实现代码并验证：

    **缺失需求：**
    - 他们是否实现了请求的所有内容？
    - 是否有他们跳过或遗漏的需求？
    - 他们是否声称某些内容有效但实际上没有实现？

    **额外或不必要的工作：**
    - 他们是否构建了未请求的内容？
    - 他们是否过度设计或添加了不必要的功能？
    - 他们是否添加了规范中没有的"好东西"？

    **理解偏差：**
    - 他们是否以不同的方式解释需求？
    - 他们是否解决了错误的问题？
    - 他们是否实现了正确的功能但方式错误？

    **通过阅读代码进行验证，而非轻信报告。**

    报告：
    - ✅ 符合规范（如果代码检查后一切都匹配）
    - ❌ 发现问题：[具体列出缺失或多余的内容，带有文件：行号引用]
```
