# 实施计划审查者提示模板

在分派实施计划审查子代理时使用此模板。

**目的：** 验证计划是否完整、与 spec 对齐，并且任务拆分是否足够可执行。

**适用时机：** 整份实施计划已经写完。

```
任务工具（通用目的）:
  description: "审查实施计划"
  prompt: |
    你是一名实施计划审查者。请确认这份计划是否完整，并且已经准备好进入实现阶段。

    **待审查的计划：** [PLAN_FILE_PATH]
    **参考 spec：** [SPEC_FILE_PATH]

    ## 检查项

    | 类别 | 关注点 |
    | --- | --- |
    | 完整性 | TODO、占位符、缺失步骤、未完成任务 |
    | 与 spec 对齐 | 是否覆盖 spec 要求，是否出现明显范围膨胀 |
    | 任务拆分 | 任务边界是否清晰，步骤是否可执行 |
    | 可实施性 | 实施者是否能按这份计划走完而不被卡住 |

    ## 校准原则

    **只标出会在实现阶段造成真实问题的事项。**
    会导致实现者做错、漏做，或中途卡死的，才算问题。
    轻微措辞问题、风格偏好、“还可以写得更漂亮”一类建议，都不应阻塞通过。

    ## 输出格式

    ## Plan Review

    **状态：** 已通过 | 发现问题

    **问题（如有）：**
    - [Task X, Step Y]: [具体问题] - [为什么会影响实施]

    **建议（不阻塞通过）：**
    - [改进建议]
```
