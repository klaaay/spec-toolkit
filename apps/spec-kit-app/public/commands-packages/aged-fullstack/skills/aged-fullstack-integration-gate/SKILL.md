# 做 aged 全栈联调与关键校验

这个 skill 用于在收尾前做最小联调校验。

## 适用场景

- 前端 service 接后端接口
- 对齐错误结构
- 跑关键验证命令
- 检查模板自举链路

## 检查项

1. 资源路由与 HTTP 方法是否符合 RESTful 语义
2. 成功响应是否统一为 `{"data": ...}`，错误响应是否统一为 `{"error": ...}`
3. 状态码是否正确区分 `200/201/400/401/403/404/500`
4. 前端 interceptor 是否正确拆包 `{"data": ...}`，service 层拿到的是否为业务数据
5. 错误结构是否稳定（`{"error": {"type": ..., "message": ...}}`）且能被前端 `normalizeApiError` 解析
6. 涉及登录或用户体系时，密码是否使用 `bcrypt` / `argon2` 加盐哈希
7. 涉及登录态时，是否明确采用 `Session` 或 `JWT`，并验证未登录、失效、无权限三类场景
8. 涉及修改、删除、敏感读取时，是否同时校验角色权限和资源归属
9. 查询是否存在明显 SQL 注入风险，是否全部走参数化或 ORM
10. 列表接口是否验证 `page >= 1`、`limit` 上限、排序字段白名单
11. 关联列表是否规避 N+1，必要时使用 `joinedload` / `selectinload`
12. 多表写操作是否放在事务中，失败时是否回滚
13. 需要热点缓存时，是否有 Redis 缓存策略或明确说明暂不引入
14. Docker 是否采用多阶段构建，运行时镜像是否只保留必要依赖
15. Nginx 是否正确指向前端构建目录，SPA 路由是否通过 `try_files` 回退
16. 涉及模板起步时，是否补跑 `node ./scripts/verify-init-project.mjs`
17. 是否至少运行：
   - `pnpm test`
   - `pnpm build:web`
18. 涉及迁移时，是否运行 `pnpm db:migrate`

## 输出要求

- 明确指出已验证项
- 明确指出未验证项
- 若契约和实现不一致，先暴露冲突，不静默绕过
