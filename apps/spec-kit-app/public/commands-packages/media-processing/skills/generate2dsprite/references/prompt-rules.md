# Prompt 规则

手写精灵 prompt 时使用本文件。

除非要与旧版生成 prompt 严格对齐，否则不要把写 prompt 交给脚本。

## 全局规则

始终遵守：

- 背景为 100% 纯色平涂品红 `#FF00FF`
- 背景无渐变
- 无文字
- 无标签
- 无 UI
- 无对话气泡
- 格数必须精确
- 格与格之间无边框或分隔线
- 各帧同一资产身份
- 各帧边界框与像素尺度一致

## 容纳规则

凡表图模式且一致性重要时，要明确写出：

- 主体须完全落在每一格内
- 肢体、特效、武器、尾尖、翼尖、法球、火星、尾迹等**不得压线或越格**
- 四边留出品红边距
- 每一帧剪影尺度相同

若不想要分离特效，写明：

- 主剪影外不要飘散的分离特效

若必须要分离特效，写明：

- 分离特效须紧挨主体、仍完全落在格内

## 视角规则

- `topdown`：大地图角色、玩家/NPC 表
- `side`：弹道、侧视单位、受击特效
- `3/4`：战斗用生物、Boss、展示待机、侧视施法者

## 角色画风（玩家与 NPC）

针对 `player` 与 `npc`：

- 16-bit RPG 大地图俯视 2D 像素风
- 略俯视的 3/4 视角
- 全身入画
- 块面清晰、深色描边的可读像素，饱和度适中
- 留出引擎渲染所需的边距

## 生物与特效画风

针对 `creature`、`spell`、`projectile`、`impact`、`summon`、`fx`：

- 剪影明确
- 体色或特效形状在战斗中一眼可读
- 姿态适合战斗或玩法阅读
- 各帧之间避免「插画感」构图漂移
- 若为人形，应明显不像玩家单位——除非用户明确要求玩家感单位

## 动作规则

### `idle`

可写：

- 中性站姿
- 轻微动态
- 重心转移或气场脉动
- 循环前最强的待机强调帧

偏好：

- 常规角色用 `2x2`
- 大型生物与展示待机用 `3x3`

### `cast`

`2x3` 施法往往是好默认：

- 预备
- 聚能
- 更强聚能
- 释放起始
- 释放峰值
- 收势或维持

### `attack`

紧凑的纯攻击表，描述：

- 蓄力
- 出手
- 随挥
- 回稳

### `hurt`

纯受击表，描述：

- 受击瞬间
- 后坐
- 踉跄
- 恢复

### `combat`

紧凑二合一表：

- 左上：攻击蓄力
- 右上：攻击出手
- 左下：受击瞬间
- 右下：受击恢复

### `projectile`

通常优先 `1x4` 或 `2x2`。

描述要点：

- 各帧同一弹道身份
- 飞行方向一致
- 形变小且可无缝循环
- 光晕或尾迹不越格

### `impact` / `explode`

通常优先 `2x2`。

描述：

- 引燃或接触
- 扩张
- 峰值爆心
- 消散或塌缩

### `walk` / `run` / `hover`

把移动方式说清楚：

- 贴地迈步
- 悬浮起伏
- 爬行
- 蛇行
- 机械滑行

## 表图专项规则

### `4x4` 主角表

约定：

- 第 1 行：朝下
- 第 2 行：朝左
- 第 3 行：朝右
- 第 4 行：朝上
- 第 1 列：中性站姿
- 第 2 列：左脚在前
- 第 3 列：再次中性
- 第 4 列：右脚在前

### `3x3` 大型待机

写明：

- 恰好 9 个等大格子的 `3x3` 网格
- 9 格边界框一致
- 主体约占每格 55%～65%
- 任何地方不得越格

### `1x4` 弹道

写明：

- 单行恰好 4 个等大格
- 各帧弹道尺寸一致
- 仅内部能量或形状脉动变化

## 组合包写法

组合包中**每个资产单独写 prompt**。

常见拆法：

- 施法单位
- 弹道
- 受击

或：

- 待机
- 战斗
- 行走

不要把不相干的资产硬塞进一张巨型表。

## 快速模板

1. 说明资产类型与表图形状  
2. 描述主体身份  
3. 按帧描述动作  
4. 重申同尺度与容纳规则  
5. 重申品红底与禁止文字
