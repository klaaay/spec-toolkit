# 分层地图约定

## 图层类型

手绘或生成的 2D RPG 场景，建议使用以下图层：

1. `base`：一张栅格图，仅含地形与地面级细节。  
2. `props`：按地图坐标锚定的透明精灵。  
3. `actors`：玩家、NPC、怪物、可拾取物与移动物体。  
4. `foreground`：可选透明精灵，必须盖在角色之上。  
5. `collision`：结构化元数据，不是像素层。  
6. `zones`：遇敌、休息、触发、出口、对话等结构化分区。  
7. `preview`：仅用于验收的压平产物。  

## 底图 Prompt 模板

生成底图时可采用如下结构：

```text
生成一张仅地面的俯视 2D 像素风 RPG 地图。
这是「仅地面底图」。
包含地形、路径、草地、水体、崖壁、地面标记与扁平承托台。
不要出现可碰撞的高物体：不要建筑、门、栅栏、路灯、树木、标牌、桶、NPC、怪物、UI 或文字。
为后续摆放道具留出明显空区。
可行走路径与分区边界要易于辨认。
```

## 道具 Prompt 模板

地图策略需要独立透明物体时，对每个道具使用 `$generate2dsprite`，prompt 直截了当：

```text
为俯视 2D 像素风 RPG 地图生成单个 <prop> 道具。
略俯视 3/4，物体完整可见、居中，深色清晰像素描边。
背景须为 100% 纯色平涂 #FF00FF 品红，无渐变、无纹理、无阴影、无地面平面。
无文字、标签、UI 或水印。
整件道具须完全在画幅内，四边留足品红边距；任何部分不得贴边或越界。
```

推荐后处理命令：

```bash
python /path/to/generate2dsprite.py process \
  --input <raw.png> \
  --target asset \
  --mode single \
  --rows 1 \
  --cols 1 \
  --cell-size 256 \
  --output-dir assets/props/<prop> \
  --fit-scale 0.9 \
  --align feet \
  --component-mode largest \
  --component-padding 8 \
  --min-component-area 200 \
  --threshold 100 \
  --edge-threshold 150 \
  --edge-clean-depth 2
```

建筑或大门等大型道具可增大 `--cell-size`。

## 道具元数据

使用明确的地图空间尺寸：

```js
const MAP_PROPS = [
  {
    id: "torii",
    imageKey: "propTorii",
    x: 836,
    y: 850,
    w: 380,
    h: 306,
    sortY: 850
  }
];
```

锚点约定：

- `x`：道具脚底/底座中心。  
- `y`：地图坐标中道具的底边。  
- `w`、`h`：地图单位下的绘制尺寸。  
- `sortY`：用于排序的 Y 深度。普通道具用底座 `y`；特殊前景行为可有意调高或调低。  

## 绘制顺序

建议顺序：

```text
底图
地面特效 / 分区微光
按 sortY 排序的可绘制物：
  道具
  玩家
  NPC
  怪物
前景叠层（仅必要时）
调试碰撞
HUD/UI
```

若某 NPC 必须始终在玩家之上，可在 Y 排序之后再画该 NPC，或为其设较高的 `sortY`。

## 碰撞元数据

保持可读、便于手改：

```json
{
  "mapSize": { "width": 1672, "height": 941 },
  "spawn": { "x": 836, "y": 782 },
  "npc": { "x": 836, "y": 390, "collisionRadius": 32, "interactRadius": 112 },
  "rest": { "x": 760, "y": 548, "radius": 122 },
  "zones": {
    "grass": { "x": 180, "y": 306, "w": 382, "h": 302 },
    "training": { "x": 1120, "y": 324, "w": 382, "h": 282 }
  },
  "walkBounds": [
    { "id": "main-courtyard", "type": "ellipse", "x": 838, "y": 548, "rx": 604, "ry": 304 }
  ],
  "blockers": [
    { "id": "torii-left-pillar", "type": "rect", "x": 704, "y": 668, "w": 52, "h": 176 }
  ]
}
```

指引：

- 阻挡用于道具**底座**，不要沿整张贴图剪影描边。  
- 用路径中心点测一遍，保证门口仍能通过。  
- 路灯、石块、树木、凹地等可用椭圆。  
- 栅栏、墙、建筑、门、桥、立柱等用矩形。  
- 仅当矩形/椭圆 walk 表现很差时再用多边形。  

## 验收清单

跑一遍或在脑中过一遍：

- 出生点可走。  
- 主路中心可走。  
- 若应能通过，门洞中心可走。  
- 门柱应挡人。  
- 栅栏阻挡但门口敞开。  
- 可交互物底座阻挡，但仍能靠近交互。  
- 遇敌/休息区可达。  
- 角色从高道具前、后走过时排序正确。  
- 压平预览与游戏内分层画面足够接近，便于目检。  

## 反模式

避免：

- 从一张完整烘焙大图里硬抠道具。  
- 在碰撞/遮挡重要时，只拿压平大图当唯一来源。  
- 把文字、标牌、UI、NPC、怪物烤进底图。  
- 道具精灵贴图像边缘。  
- 把透明 PNG 的外接框自动当成碰撞。  
- 只改美术却不更新碰撞与关键点测试。
