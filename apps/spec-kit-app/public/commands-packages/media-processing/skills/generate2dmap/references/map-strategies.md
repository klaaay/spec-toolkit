# 地图策略选型

在**仍能管住玩法**的前提下，选最轻的结构。

## 单张烘焙图

适用于：

- 场景小、静态、偏装饰，或固定屏宽
- 无碰撞，或碰撞很粗，用简单隐形形状即可
- 没有角色需要走到高物体后方
- 用户更看重速度而非日后可编辑性
- 地图作为菜单背景、战斗背板、标题、过场或快速原型

交付：

- 一张地图图
- 可选 prompt 文件
- 可选简单碰撞形状
- 直接改代码接入

**不要**仅仅因为技能支持拆道具就去硬拆。

## 分层底图 + 道具

适用于：

- 需要较精细的阻挡
- 角色要从物体前、后走过
- 物体可交互、可动画、可复用、可破坏，或日后常改图
- 用户抱怨碰撞不准或前后景叠乱
- 场景是 RPG 探索图、城镇、地城、野外、神社、民宅室内或战术场地

交付：

- 仅地面的底图
- 通过 `$generate2dsprite` 生成的道具精灵，或由项目另行提供
- 道具摆放元数据
- 碰撞/分区元数据
- 压平预览供验收
- 带绘制排序的代码接入

生成类道具精灵用 `$generate2dsprite`。不要仅因存在该策略就拆道具；**只有**在独立碰撞、遮挡、交互、复用、动画或后续改图能带来收益时，才选分层道具。

## 瓦片地图 / 图块集

适用于：

- 现有引擎已用 Tiled、Phaser 瓦片地图、Godot TileMap、Unity Tilemap 等
- 用户要瓦片、图块集、瓦片碰撞、自动瓦片，或与地图编辑器互通
- 格点级编辑、程序化生成或超大地图很关键

按引擎交付：

- 图块集图片
- 瓦片地图 JSON/TMX/引擎原生数据
- 瓦片碰撞层
- 对象/触发层

除非用户要求，否则**不要**把纯生图流程硬扭成瓦片工作流。

## 混合方案

适用于：

- 大部分场景可以烘焙，只有少数物体需要单独控制
- 只有前景遮挡或可交互物需要独立精灵
- 项目已有烘焙图，仅一两处特别难受

交付：

- 烘焙背景或底图
- 仅必要的道具/前景叠层
- 针对性碰撞元数据

## 升级启发

从简到繁：

1. 单图  
2. 单图 + 碰撞元数据  
3. 混合：烘焙 + 少量道具/叠层  
4. 完整分层：底图 + 道具  
5. 瓦片地图——仅当引擎/编辑器需求成立时
