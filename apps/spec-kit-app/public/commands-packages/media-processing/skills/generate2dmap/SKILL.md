# Generate2dmap（2D 地图生成）

## 概述

选用并搭建**能满足玩法所需的最简** 2D 地图结构。本技能是地图编排器：判断最终产物应是单张烘焙图、烘焙图 + 碰撞元数据、混合地图、分层底图 + 道具，还是瓦片地图。

- **简单或偏装饰**的地图 → 单张烘焙图。
- **碰撞、遮挡、交互、复用、动画**重要 → 分层资源。
- 现有引擎或用户明确要求**按瓦片编辑** → 瓦片地图。

仅在所选策略**确实需要可复用的透明道具**（常见于分层或混合方案）时，再调用 `$generate2dsprite`。不要因为「本技能能挂道具」就硬拆道具。

## 工作流

1. **摸清目标游戏**
   - 地图尺寸、镜头比例、渲染循环、资源加载方式、支持的碰撞形状、出生点/休息点/NPC/遇敌区等数据，以及调试碰撞工具。
   - 保留引擎既有画风与坐标系。

2. **选定地图策略**
   - **单张烘焙图**：适合简单地图、菜单/背景场景、固定战场、原型，以及碰撞很粗或很少的游戏。
   - **分层地图**：适合 RPG 探索、精细碰撞、高物体、前后遮挡、跨区复用道具、可动画道具或可交互物。
   - **瓦片地图**：引擎已在用瓦片、用户要瓦片编辑/导出，或格点级关卡设计很关键时。
   - **混合**：大块场景仍可烘焙，仅把少数需要单独控制的物体（叠层、道具或碰撞体）拆出来。
   - 策略不明显时读 [references/map-strategies.md](references/map-strategies.md)。

3. **产出对应资源集**
   - **单张烘焙**：生成或改好一张完整地图图，直接接入。
   - **单张烘焙 + 玩法约束**：完整地图图 + 碰撞或分区元数据。
   - **混合**：大部分场景仍烘焙，只把需要碰撞、遮挡、交互、复用或动画的对象拆出。
   - **分层**：只做地面的底图 + 生成的道具。道具资产用 `$generate2dsprite`，再按地图坐标摆放。新建分层地图或大地图改版前读 [references/layered-map-contract.md](references/layered-map-contract.md)。
   - **瓦片地图**：按项目既有瓦片格式来；仅当运行引擎支持时，再新建图块集/地图 JSON。

4. **搭建碰撞与分区**
   - 优先用简单结构化形状：矩形、椭圆、多边形。
   - **碰撞数据与美术文件分离。**
   - 只加这张地图真正需要的玩法元数据：可行走边界、阻挡、遇敌区、休息区、NPC 碰撞半径、出生点、战斗返回点、出口、触发器等。
   - 单张烘焙图若玩法不要求精度，碰撞可以粗或很少。
   - 分层地图对关键点做程序校验：出生点可走、路径中心可走、道具底座阻挡、门口保持畅通。

5. **验收与迭代**
   - 使用分层资源时，输出一张**压平预览**。
   - 检查地图尺度、镜头构图、与文字/UI 重叠、碰撞、路径可读性、键鼠/手柄移动体验。
   - 分层道具质量差或贴边时，**重生成或重后处理道具**，不要从烘焙大图里硬抠。

## 决策规则

- 优先选**能撑起需求的最简单**地图形态。
- 地图偏装饰、很小、固定屏宽、或只需大块碰撞 → 单张烘焙。
- 物体需要阻挡、遮挡、交互、动画、复用或独立改图 → 分层道具。
- **纯地面细节**画进底图。
- 烘焙图在碰撞或遮挡上别扭时，**只把有问题的物体改成道具**，不必全盘重做。
- 仅当游戏已在用瓦片工具链，或用户明确要求瓦片编辑器/导出时，才上瓦片集。
- **地图美术里不要生成可读文字**；标签和 UI 放在代码层。

## 预期交付物

**简单烘焙场景**：

- `assets/map/<name>.png`
- 可选 `<name>.prompt.txt`
- 玩法需要时的可选碰撞元数据
- 加载/使用该图的代码改动

**可玩的分层场景**：

- `assets/map/<name>-base.png`
- `assets/map/<name>-base.prompt.txt`
- 通过 `$generate2dsprite` 生成的各道具目录
- `assets/props/<prop-name>/prop.png` 以及 `pipeline-meta.json`
- 按目标游戏格式书写的道具摆放元数据
- `assets/map/<name>-layered-preview.png`：压平后的最终预览，供验收与展示
- 碰撞类元数据，例如 `data/collision-map.json`
- 加载底图、加载道具图、按 Y 深度排序绘制道具/角色、使用碰撞元数据的代码改动

## 校验

务必按所选策略核对：

- 地图文件存在且尺寸符合预期
- 使用道具时，透明道具 PNG 带 alpha
- 使用生成道具时，`pipeline-meta.json` 中不应出现 `edge_touch_frames`
- 游戏侧 JavaScript/TypeScript 能解析通过
- 存在碰撞 JSON 时能正确解析
- 需要精细碰撞时，关键点测试通过
- 预览或实机画面在目标镜头尺度下观感统一
