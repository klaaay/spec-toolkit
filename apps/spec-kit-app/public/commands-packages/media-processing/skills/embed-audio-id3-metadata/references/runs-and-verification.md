# 多遍运行与验收说明

## 为什么要多遍（`--passes`）

网易云音乐 Web API 在短时间大量请求下会出现：

- 歌词接口返回空 `lrc`；
- 搜索或详情偶发超时/不完整 JSON。

脚本虽已对单次请求做了有限重试、并在单文件内对歌词匹配做了第二轮尝试，**仍无法保证单遍 100% 填满**。对整库扫描，应用 **`--passes 2`～`4`**，并在遍与遍之间保留短暂间隔（脚本内约 2 秒），可显著降低残留 `no_lyrics`。

## 推荐流程

1. 安装依赖：`pip install mutagen zhconv`
2. 执行：  
   `python3 scripts/embed_mp3_metadata.py "<根目录>" --passes 3`
3. 保存日志：  
   `... 2>&1 | tee embed_run.log`
4. 统计缺词：  
   `grep -c 'no_lyrics' embed_run.log`  
   若大于 0，可再执行一遍（仅再跑 1～2 遍通常即可），或隔一段时间再跑（避开限流）。

## 日志字段含义（节选）

| 片段              | 含义                       |
| ----------------- | -------------------------- |
| `local_cover`     | 使用了同文件夹内图片       |
| `remote_cover`    | 使用了拉取的在线封面       |
| `no_cover`        | 未找到封面                 |
| `lyrics_official` | 平台词（`lyricUser` 为空） |
| `lyrics_fan`      | 用户上传词                 |
| `no_lyrics`       | 该遍仍未写入歌词           |

## 根目录解析顺序

1. 命令行第一个参数（音乐库根目录）
2. 环境变量 `MUSIC_EMBED_ROOT`
3. 脚本所在目录（适用于把脚本放在库根目录旁时）
